## 已实现功能

- 全部题目练习。
- 错题练习。
- 模拟考试。
- 支持打开网站后选择科目。
- 全部题目、错题练习、模拟考试均提供答题卡，可点击题序跳转。
- 单选题使用单选框。
- 多选题使用复选框。
- 判断题使用单选框，选项为“对/错”。
- 训练模式中，单选题选择后立即判题并显示正确答案。
- 训练模式中，判断题选择后立即判题并显示正确答案。
- 训练模式中，多选题保留提交按钮，提交后显示正确答案。
- 支持“上一题”和“下一题”。
- 错题自动进入错题本。
- 错题记录保存在浏览器 `localStorage` 中。
- 错题练习中，同一道错题连续答对 `2` 次后自动移出错题集。
- 错题本显示每题连续答对进度，例如 `已连续答对 1 / 2 次`。
- 可清空错题。
- 可点击错题本中的题目进入错题练习。

## 模拟考试规则

- 每套模拟考试从当前科目随机抽取 `140` 道。
- 模拟考试按“单选题、多选题、判断题”的顺序排列。
- `国家安全` 模拟考试配比：单选 `112` 题，多选 `28` 题，约为 `80% / 20%`。
- `思政` 模拟考试配比：单选 `56` 题，多选 `28` 题，判断 `56` 题，约为 `40% / 20% / 40%`。
- 如果某个题型题量不足，会从当前科目的剩余题目中补足到 `140` 题。
- 一套模拟题内不会出现重复题目。
- 满分 `100` 分。
- 模拟考试中，单选题和判断题选择后自动保存并进入下一题。
- 模拟考试中，多选题保留保存按钮，保存后进入下一题。
- 用户需要全部作答后才能交卷。
- 如果有未答题，交卷时会自动跳转到第一道未答题。
- 交卷后显示：
  - 分数。
  - 试题数量。
  - 答对数量。
  - 答错数量。
  - 错题序号。
- 用户点击错题序号可以跳转到模拟考试中的对应题目。
- 模拟考试中答错的题会自动整理进错题集。

## 本地存储

当前使用浏览器 `localStorage` 保存错题状态：

- `quiz-wrong-question-ids-v3:<科目名>`：错题题号集合。
- `quiz-wrong-correct-streak-v2:<科目名>`：错题练习连续答对次数。
- `quiz-current-subject-v1`：上次选择的科目。

兼容旧版本错题键：

- `quiz-wrong-question-ids-v1`
- `quiz-wrong-question-ids-v2`
- `quiz-wrong-correct-streak-v1`

旧版全局错题记录只迁移到默认科目，新增科目不会读取旧版全局错题。

注意：错题记录保存在使用者自己的浏览器中，不会随着文件夹分发同步给别人。

## 设计原则

- 保持纯静态网站，便于把整个 `D:\答题` 文件夹发给别人。
- 不依赖后端、数据库、Node.js 或 Python 运行环境。
- 尽量保证双击 `index.html` 即可使用。
- 题库通过 `questions.js` 直接加载，避免本地浏览器限制读取 JSON 文件。
- 基础无障碍支持：按钮文字明确、反馈不只依赖颜色、支持键盘操作、移动端可用。

## 重要实现说明

- `app.js` 是主要状态机，当前模式通过 `mode` 控制：
  - `all`：全部题目练习。
  - `wrong`：错题练习。
  - `exam`：模拟考试。
- 模拟考试题目保存在 `examQuestions`。
- 模拟考试作答记录保存在 `examAnswers`。
- 模拟考试交卷状态保存在 `examSubmitted`。
- 错题集合保存在 `wrongIds`。
- 连续答对次数保存在 `wrongCorrectStreak`。
- 当前科目保存在 `currentSubject`。
- 当前科目题库保存在 `questions`。
- 答题卡状态来自训练结果 `practiceResults` 或模拟考试答案 `examAnswers`。
